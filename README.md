# english-quiz

Single-file PWA + Claude-Code-driven coaching system for 5 Russian-speaking family
members. Live at `artemseliverstov.github.io/english-quiz`. Operating instructions:
`CLAUDE.md` (routing) → `references/` (rules). This file is the **repo map**.

## Folder contract

| Folder | Purpose | Regime |
|---|---|---|
| `docs/` | Design doctrine — why the system is shaped this way. Slow-changing. | authored |
| `references/` | Operational KB — schemas, rules, checklists, logs. | authored |
| `plans/` | Execution specs. Every plan carries a `**Status**:` line. | authored |
| `progress/` | Learner-facing trackers. | **generated** — regenerated by `stats-review`, never hand-edited |
| `diagnostics/` | Assessment instruments + responses + scored results + study packs. | authored + learner data |
| `library_drafts/` | Authoring source for the Coach exercise library (per-player drill JSONs; published to Firestore `exercises_library` via `tools/push_library.js`). | authored |
| `tools/` | Live CLI scripts + their state files (`data-integrity-baseline.json`, `mistake_verdicts.json`). Nothing else. | code |
| `worker/` | Cloudflare Worker (live-AI proxy + `/v1/audio`). | code |
| `ref/` | Player-facing grammar reference page (📖 link in the app header). | code |
| `archive/` | Superseded artifacts (HTML KB snapshots, ingested data). Tracked. | archive |
| `audits/` | Dated bank-audit reports from `tools/bank_audit.js`. Tracked, one per run. | generated |
| `.claude/` | Skills, settings, hooks. | code |
| `.github/` | CI + Pages artifact deploy (app files only) + nightly Firestore backup (to the private `english-quiz-backups` repo). | code |
| local-only | `backups/` (manual snapshots), `Interview/` (audio — personal). | gitignored |

Root files: `index.html` (the whole app, questions inline — by design), `sw.js`,
`manifest.json` + icons, `firestore.rules` + firebase config, `mathsprint.html`
(standalone mental-arithmetic sprint game — math track, outside the quiz engine;
shared family leaderboard in `math_sprint_scores`), `memory.html` (standalone
working-memory/attention trainer — cognitive track, outside the quiz engine;
family board in `memory_trainer_scores`).

## House rules

1. **One-off scripts never enter the repo.** They run from the Claude session
   scratchpad and die there; only their *outcome* is committed (code change,
   Firestore write, version-log entry). Data artifacts worth keeping → `archive/`.
2. **Naming**: markdown = kebab-case (README/CLAUDE/SKILL conventions excepted);
   js/json = snake_case; per-player files prefixed `{player}_`; dates ISO.
3. **Archive, don't delete**: docs retire into their folder's `archive/`
   (`references/archive/`, `plans/archive/`); heavyweight or cross-format artifacts
   into root `archive/`. `tools/check_doc_pointers.js --strict` (CI) keeps referrers
   honest.
4. **Generated files** carry a "generated — don't hand-edit" banner + `Last refresh`
   date. A new top-level folder requires a row in this table in the same commit.
5. **Personal data stays out** — exports, recordings, transcripts never live inside
   the repo folder (even untracked). `Interview/` is the one sanctioned local-only
   exception.

Word caps, doc style: `references/doc-style.md`. Deploy: `deploy-build` skill.
